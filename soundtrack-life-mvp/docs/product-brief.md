# 产品方案：人生原声机

## 项目定位

人生原声机是一个 AI 音频编排产品。它不直接替代音乐创作，而是把用户的生活轨迹、情绪状态和长音频内容，编排成一段可播放、可解释、可分享的私人声音体验。

核心价值：

- 对音乐用户：把“今天听什么”从搜索/刷歌单变成一句话生成。
- 对长音频用户：把“没时间听完”变成“听重点、按问题听、按章节听”。
- 对平台：提升曲库消费、长音频消费和分享传播。

## 用户场景

### 场景 1：今天需要一张属于自己的 BGM

用户晚上打开 App，输入今天的经历：

```text
今天面试失败了，下午淋了雨，晚上不想太丧，想慢慢恢复一点。
```

AI 输出：

- 今日标题：《雨后慢慢亮起来》
- 情绪路径：失落 -> 接住自己 -> 轻微恢复
- 4 段时间线：雨中、低谷、安静、重新出发
- 每段推荐歌曲
- 每段 DJ 旁白
- 分享卡片

### 场景 2：播客太长，只想听对自己有用的部分

用户粘贴播客转写或选择平台内长音频：

```text
帮我找出这期播客里关于“AI 音乐版权”的重点。
```

AI 输出：

- 章节地图
- 关键片段时间戳
- 3 分钟速听版
- 相关背景解释
- 可继续追问

### 场景 3：从长音频延伸到音乐

用户听完一段关于“城市夜跑”的播客，系统生成：

- 夜跑歌单
- 播客提到的音乐关键词
- 适合延伸收听的歌曲和节目

## 产品原则

- 优先使用真实曲库推荐，避免在原型阶段陷入完整歌曲生成。
- AI 的职责是理解、编排、解释和陪伴。
- 输出必须可听、可看、可分享。
- 每个结果都要有“为什么推荐”，让体验显得聪明而不是随机。

## MVP 功能范围

必须做：

- 生活轨迹输入
- 情绪/场景结构化
- BGM 推荐结果
- DJ 旁白生成
- 长音频文本章节化
- 长音频问答导航
- Web 可视化结果页

可以模拟：

- 真实音乐平台曲库接口
- 用户历史听歌画像
- 音频播放
- TTS 真实口播

暂不做：

- 端到端音乐生成
- 真正的版权曲播放
- 复杂账号体系
- 多人社交关系链

## 差异化亮点

1. 从“推荐歌”升级为“生成一段人生叙事”。
2. 同时服务音乐和长音频，符合赛题范围。
3. 适合中文语境，能理解“别太 emo”“下班松弛感”“凌晨自救”等表达。
4. 可用音乐平台资产放大：曲库、评论、歌单、播客、有声书。

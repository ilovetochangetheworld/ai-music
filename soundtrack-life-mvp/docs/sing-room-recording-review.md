# AI 声友局：录音与唱后评价设计

## 目标

把主体验补成一个可演示闭环：陪唱与救场结束后，用户能立即回放自己的演唱，并收到具体、温和、可追溯的反馈。

## 体验原则

- 录音前明确说明用途；MVP 录音只留在当前浏览器会话，不上传。
- 用户可回放、下载或立即删除录音。
- 不展示制造焦虑的总分，不评价天赋。
- 每条评价必须能由麦克风检测帧或事件日志证明。
- 数据不足时显示“数据不足”，不补造结论。

## 当前评价维度

| 维度 | 数据来源 | 表达边界 |
| --- | --- | --- |
| 开口参与 | 演唱开始/停止事件、可唱歌词时长 | 只表示参与比例 |
| 声音清晰 | 有效演唱帧的 pitch clarity | 受麦克风和环境影响 |
| 音高连续 | 相邻有效音高帧的半音变化 | 不等同于音准正确 |
| 救场接回 | RESCUE_STARTED / RESCUE_ENDED | 只描述接回行为 |

当前版本不提供“音准正确率”和“节奏准确率”，因为尚未建立经过校验的参考旋律与逐拍对齐数据。

## 技术流程

```text
用户点击开始
  -> 同时唤醒 AudioContext 与申请麦克风
  -> 环境校准和倒数
  -> MediaRecorder 录制原始麦克风
  -> 每 80ms 记录检测帧
  -> 结束时生成事实指标与温和文案
  -> 唱后页回放 / 高光句回放 / 下载 / 删除
```

舞台反馈使用互动预算控制：安静模式不自动鼓掌，适中模式最多一次，热闹模式最多两次；两次掌声之间至少间隔 18 秒。掌声只由完整唱过副歌句这一事实触发。

## 后续增强

1. 从授权主唱轨提取参考旋律，加入移调容错后的音准贴合度。
2. 根据歌词字级或音符级时间轴计算节奏贴合度。
3. 用 IndexedDB 保存用户主动保留的录音。
4. 在用户明确同意后上传录音，生成更完整的异步分析。

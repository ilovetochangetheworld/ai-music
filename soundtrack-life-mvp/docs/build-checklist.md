# 开发 Checklist

## 0. 初始化

- [ ] 创建前端项目
- [ ] 加入基础样式方案
- [ ] 配置环境变量
- [ ] 准备 mock 模式开关
- [ ] 拷贝 `samples/songs.json`
- [ ] 拷贝 `samples/podcast-transcript.json`

## 1. 首页

- [ ] 做产品标题和一句话说明
- [ ] 做两个入口 tab：日轨 BGM / 长音频速听
- [ ] 日轨 BGM 输入框支持多行文本
- [ ] 长音频输入支持粘贴 transcript
- [ ] 加入示例按钮，一键填入 demo 文案

验收：

- 不接 API 也能点击示例进入结果页。

## 2. 日轨 BGM

- [ ] 实现 `parseLifeText(text)`，调用 LLM 或返回 mock JSON
- [ ] 实现 `matchSongs(scene, songs)`，基于 mood/scene/energy 匹配
- [ ] 实现 `generateNarration(scenes)`，调用 LLM 或使用 mock
- [ ] 做情绪曲线组件
- [ ] 做场景时间线组件
- [ ] 做歌曲卡片组件
- [ ] 做 DJ 旁白组件
- [ ] 做模拟播放器状态

验收：

- 输入一句生活描述，可以生成 3-5 个场景。
- 每个场景有歌、有理由、有旁白。
- 页面看起来像一个真的音乐产品，而不是 JSON 预览器。

## 3. 长音频速听

- [ ] 实现 `analyzeTranscript(transcript)`
- [ ] 做章节地图组件
- [ ] 做 3 分钟速听路线
- [ ] 做 15 分钟精听路线
- [ ] 实现 `askAudioCoach(question)`
- [ ] 做问题输入框
- [ ] 做推荐片段列表
- [ ] 做时间戳跳转的模拟交互

验收：

- 用户问“只听版权风险部分”，页面返回 1-3 个时间片段。
- 每个片段有标题、时间、推荐理由。

## 4. 分享卡片

- [ ] 生成今日标题
- [ ] 展示情绪路径
- [ ] 展示 3 首代表歌曲
- [ ] 展示一句分享文案
- [ ] 支持下载图片，可选

验收：

- 结果页能展示一张适合截图的分享卡。

## 5. 演示稳定性

- [ ] 所有 LLM 调用都有 mock fallback
- [ ] loading 状态不超过 10 秒
- [ ] API 失败时使用本地示例数据
- [ ] 移动端布局不崩
- [ ] 准备两组固定输入

## 推荐优先级

P0：

- 首页
- 日轨 BGM 完整链路
- mock 曲库匹配
- 结果页
- 演示脚本

P1：

- 长音频章节化
- 长音频问答
- 分享卡片

P2：

- TTS
- 真实播放器
- 用户历史偏好
- 图片海报下载

## 可复用 Demo 输入

### 日轨 BGM

```text
今天早上通勤很堵，上午做了一个重要汇报，下午被老板夸了，晚上想一个人散步，不想太兴奋，想慢慢放松。
```

### 情绪更强的版本

```text
今天面试失败了，下午下雨，回家的路上突然觉得很累。但我不想听太惨的歌，想要一点点被接住，然后慢慢恢复。
```

### 长音频问题

```text
只听关于 AI 音乐版权风险的部分。
```
